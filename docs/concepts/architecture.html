Core Architecture

Understand the modular architecture that powers Agents Hub and how its components work together to create a flexible, powerful agent orchestration framework.

Architectural Overview

Agents Hub is built on a modular, extensible architecture designed to support a wide range of agent-based applications. The architecture follows these key principles:

  • Modularity: Components are designed to be interchangeable and independently upgradable
  • Extensibility: The framework can be extended with custom components
  • Provider-agnostic: Support for multiple LLM providers
  • Cognitive-first: Advanced cognitive capabilities are central to the design
  • Multi-agent: Built to support complex agent workforces

Core Components

The Agents Hub architecture consists of several core components that work together:

Agent Core

The foundational component that manages the agent lifecycle, handles interactions, and coordinates other components.

LLM Providers

Adapters for different language model providers (OpenAI, Claude, etc.) that handle API communication and response parsing.

Cognitive Architecture

Advanced reasoning capabilities including metacognition, multiple reasoning strategies, and learning mechanisms.

Memory System

Storage and retrieval mechanisms for conversation history, knowledge, and agent state.

Tool System

Framework for defining, managing, and executing tools that extend agent capabilities.

Orchestration

Mechanisms for coordinating multiple agents in a workforce, including communication and task delegation.

Moderation

Content filtering and safety mechanisms to ensure appropriate agent behavior.

Monitoring

Observability and analytics for tracking agent performance and behavior.

Component Interactions

The components in Agents Hub interact through well-defined interfaces:

└─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────────┘

When an agent processes a request, the typical flow is:

  1. The Agent Core receives the input and initializes the processing pipeline
  2. The Memory System is queried for relevant context from past interactions
  3. The Cognitive Architecture determines the reasoning approach
  4. The Tool System identifies and executes any required tools
  5. The LLM Provider generates a response based on the context, reasoning, and tool outputs
  6. The Moderation system checks the response for policy compliance
  7. The Monitoring system logs the interaction
  8. The Agent Core returns the final response

Agent Types

Agents Hub provides several agent types built on this architecture:

Basic Agent

The standard agent implementation that provides core functionality:

from agents_hub import Agent
from agents_hub.llm.providers import OpenAIProvider

agent = Agent(
    name="assistant",
    llm=OpenAIProvider(api_key="your-api-key"),
    system_prompt="You are a helpful assistant."
)

Cognitive Agent

An enhanced agent with advanced cognitive capabilities:

from agents_hub import CognitiveAgent
from agents_hub.cognitive import CognitiveArchitecture

cognitive_agent = CognitiveAgent(
    name="cognitive_assistant",
    llm=llm,
    cognitive_architecture=CognitiveArchitecture(),
    system_prompt="You are an assistant with advanced reasoning capabilities."
)

Agent Workforce

A team of specialized agents working together:

from agents_hub import AgentWorkforce, Agent

# Create specialized agents
researcher = Agent(name="researcher", llm=llm, system_prompt="You research information.")
writer = Agent(name="writer", llm=llm, system_prompt="You write content.")
editor = Agent(name="editor", llm=llm, system_prompt="You edit and improve content.")

# Create a workforce
workforce = AgentWorkforce(
    agents=[researcher, writer, editor],
    orchestrator_agent=researcher
)

Custom Tools

Create new tools to extend agent capabilities:

from agents_hub.tools.base import BaseTool
from pydantic import BaseModel, Field

class WeatherInput(BaseModel):
    location: str = Field(..., description="The city and state, e.g. San Francisco, CA")

class WeatherTool(BaseTool):
    name = "get_weather"
    description = "Get the current weather in a given location"
    input_schema = WeatherInput

    async def _execute(self, location: str) -> str:
        # Implement weather API call
        weather_data = await self._fetch_weather(location)
        return f"The weather in {location} is {weather_data['condition']} with a temperature of {weather_data['temperature']}°F."

Custom Memory Systems

Create custom memory backends:

from agents_hub.memory.base import BaseMemory
from typing import List, Dict, Any

class CustomMemory(BaseMemory):
    def __init__(self, connection_string):
        self.connection_string = connection_string
        self.client = self._initialize_client()

    def _initialize_client(self):
        # Initialize connection to your custom storage
        pass

    async def add(self, message: Dict[str, Any]) -> None:
        # Store a message in your custom storage
        await self.client.store(message)

    async def get_recent(self, n: int = 10) -> List[Dict[str, Any]]:
        # Retrieve recent messages
        return await self.client.get_recent(n)

    async def search(self, query: str, n: int = 5) -> List[Dict[str, Any]]:

Design Patterns

Agents Hub uses several design patterns to create a flexible, maintainable architecture:

Dependency Injection

Components are injected into agents rather than created internally, allowing for easy substitution and testing:

# Dependency injection example
agent = Agent(
    name="assistant",
    llm=OpenAIProvider(api_key="your-api-key"),
    memory=PostgreSQLMemory(connection_string="..."),
    tools=[WebSearchTool(), CalculatorTool()],
    moderator=ContentModerator(api_key="...")
)

Strategy Pattern

Different strategies can be selected for various components:

# Different memory strategies
in_memory = InMemoryMemory()
postgres_memory = PostgreSQLMemory(connection_string="...")

# Different LLM provider strategies
openai_llm = OpenAIProvider(api_key="...")
claude_llm = ClaudeProvider(api_key="...")

Observer Pattern

Monitoring components observe agent behavior without affecting core functionality:

from agents_hub.monitoring import LangfuseMonitor

# Add monitoring without changing agent behavior
monitor = LangfuseMonitor(
    api_key="your-langfuse-api-key",
    public_key="your-langfuse-public-key"
)

agent = Agent(
    name="assistant",
    llm=llm,
    monitor=monitor
)

Conclusion

The modular architecture of Agents Hub provides a flexible, extensible foundation for building sophisticated agent systems. By understanding how these components work together, you can leverage the full power of the framework and extend it to meet your specific needs.
