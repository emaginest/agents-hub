Agents

Explore the different types of agents in Agents Hub, their capabilities, and how to configure them for various use cases.

What is an Agent?

In Agents Hub, an agent is an autonomous entity powered by a language model (LLM) that can:

  • Process natural language inputs
  • Generate contextually relevant responses
  • Use tools to interact with external systems
  • Maintain memory of past interactions
  • Follow instructions defined in a system prompt
  • Collaborate with other agents in a workforce

Agents are the core building blocks of the Agents Hub framework. They encapsulate the intelligence, capabilities, and behaviors needed to perform tasks and interact with users and other agents.

Agent Types

Agents Hub provides several types of agents with different capabilities:

Basic Agent

The standard agent implementation that provides core functionality:

from agents_hub import Agent
from agents_hub.llm.providers import OpenAIProvider

agent = Agent(
    name="assistant",
    llm=OpenAIProvider(api_key="your-api-key"),
    system_prompt="You are a helpful assistant."
)

response = await agent.run("What is the capital of France?")

Basic agents are suitable for straightforward tasks like answering questions, providing information, or performing simple actions using tools.

Cognitive Agent

An enhanced agent with advanced cognitive capabilities:

from agents_hub import CognitiveAgent
from agents_hub.llm.providers import ClaudeProvider
from agents_hub.cognitive import CognitiveArchitecture

cognitive_agent = CognitiveAgent(
    name="cognitive_assistant",
    llm=ClaudeProvider(api_key="your-api-key"),
    cognitive_architecture=CognitiveArchitecture(
        metacognition_enabled=True,
        reasoning_mechanisms=["deductive", "inductive", "abductive"]
    ),
    system_prompt="You are an assistant with advanced reasoning capabilities."
)

response = await cognitive_agent.run("What are the ethical implications of AI in healthcare?")

Cognitive agents are ideal for complex tasks that require sophisticated reasoning, self-reflection, and learning. They excel at:

  • Solving complex problems
  • Analyzing nuanced situations
  • Providing well-reasoned explanations
  • Adapting to new information
  • Handling uncertainty and ambiguity

For more details on cognitive agents, see the Cognitive Architecture section.

Specialized Agents

You can create specialized agents for specific roles by configuring their system prompts, tools, and other properties:

# Research agent
research_agent = Agent(
    name="researcher",
    llm=llm,
    tools=[WebSearchTool(), DocumentReaderTool()],
    system_prompt="""You are a research assistant specialized in finding and analyzing information.
    Your goal is to provide accurate, well-sourced information on any topic.
    Always cite your sources and be transparent about the limitations of your knowledge."""
)

# Customer support agent
support_agent = Agent(
    name="support",
    llm=llm,
    tools=[KnowledgeBaseTool(), TicketCreationTool()],
    system_prompt="""You are a customer support representative for Acme Inc.
    Your goal is to help customers resolve their issues efficiently and professionally.
    If you cannot resolve an issue, create a support ticket for further assistance."""
)

# Code assistant
code_agent = Agent(
    name="coder",
    llm=llm,
    tools=[CodeAnalysisTool(), GitTool()],
    system_prompt="""You are a coding assistant specialized in Python development.
    Your goal is to help developers write clean, efficient, and well-documented code.
    Always explain your code suggestions and consider best practices."""
)

Agent Configuration

Agents can be configured with various components and settings:

LLM Providers

Agents can use different LLM providers:

# OpenAI
from agents_hub.llm.providers import OpenAIProvider
openai_agent = Agent(
    name="openai_agent",
    llm=OpenAIProvider(
        api_key="your-openai-api-key",
        model="gpt-4"
    )
)

# Claude
from agents_hub.llm.providers import ClaudeProvider
claude_agent = Agent(
    name="claude_agent",
    llm=ClaudeProvider(
        api_key="your-claude-api-key",
        model="claude-3-opus-20240229"
    )
)

# Google
from agents_hub.llm.providers import GeminiProvider
gemini_agent = Agent(
    name="gemini_agent",
    llm=GeminiProvider(
        api_key="your-gemini-api-key",
        model="gemini-pro"
    )
)

# Ollama (local)
from agents_hub.llm.providers import OllamaProvider
ollama_agent = Agent(
    name="ollama_agent",
    llm=OllamaProvider(
        host="http://localhost:11434",
        model="llama3"
    )
)

Memory

Agents can use different memory systems:

# In-memory (default)
from agents_hub.memory import InMemoryMemory
agent_with_memory = Agent(
    name="agent_with_memory",
    llm=llm,
    memory=InMemoryMemory()
)

# PostgreSQL with vector search
from agents_hub.memory import PostgreSQLMemory
agent_with_postgres = Agent(
    name="agent_with_postgres",
    llm=llm,
    memory=PostgreSQLMemory(
        connection_string="postgresql://user:password@localhost:5432/agents_hub",
        table_name="agent_memory",
        embedding_model="text-embedding-ada-002"
    )
)

Tools

Agents can use various tools to extend their capabilities:

from agents_hub.tools.standard import (
    WebSearchTool,
    CalculatorTool,
    WeatherTool,
    WikipediaTool
)

agent_with_tools = Agent(
    name="agent_with_tools",
    llm=llm,
    tools=[
        WebSearchTool(api_key="your-search-api-key"),
        CalculatorTool(),
        WeatherTool(api_key="your-weather-api-key"),
        WikipediaTool()
    ]
)

Moderation

Agents can use moderation to ensure safe and appropriate responses:

from agents_hub.moderation import ContentModerator

agent_with_moderation = Agent(
    name="agent_with_moderation",
    llm=llm,
    moderator=ContentModerator(
        api_key="your-moderation-api-key",
        content_policy={
            "profanity": "block",
            "hate_speech": "block",
            "sexual_content": "block",
            "violence": "warn"
        }
    )
)

Monitoring

Agents can be monitored for performance and behavior:

from agents_hub.monitoring import LangfuseMonitor

agent_with_monitoring = Agent(
    name="agent_with_monitoring",
    llm=llm,
    monitor=LangfuseMonitor(
        api_key="your-langfuse-api-key",
        public_key="your-langfuse-public-key"
    )
)

Agent Lifecycle

Agents in Agents Hub follow a lifecycle that includes:

Initialization

When an agent is created, it initializes its components (LLM provider, memory, tools, etc.) and prepares for interaction.

Processing

When the run() method is called, the agent processes the input through several steps:

  1. Retrieve relevant context from memory
  2. Prepare the prompt with system instructions, context, and user input
  3. Send the prompt to the LLM provider
  4. Parse the response and execute any tool calls
  5. Apply moderation if configured
  6. Store the interaction in memory
  7. Return the final response

Tool Execution

When an agent uses tools, it follows this process:

  1. The LLM identifies the need for a tool and specifies the tool name and parameters
  2. The agent finds the appropriate tool in its tool set
  3. The agent executes the tool with the provided parameters
  4. The tool result is returned to the agent
  5. The agent incorporates the tool result into its response

Memory Management

Agents manage their memory by:

  1. Storing user inputs and agent responses
  2. Retrieving relevant past interactions when processing new inputs
  3. Optionally performing semantic search to find related information

Best Practices

Here are some best practices for working with agents:

System Prompts

Design effective system prompts that:

  • Clearly define the agent's role and purpose
  • Specify the tone and style of communication
  • Provide guidelines for handling different types of requests
  • Include any domain-specific knowledge or instructions
  • Set boundaries for what the agent should and shouldn't do

Tool Selection

Choose appropriate tools based on the agent's purpose:

  • Only include tools that the agent needs for its specific role
  • Provide clear descriptions for each tool to help the LLM understand when to use them
  • Consider the security implications of each tool

LLM Selection

Choose the right LLM for your use case:

  • For complex reasoning tasks, use more powerful models like GPT-4 or Claude 3 Opus
  • For simpler tasks, more efficient models like GPT-3.5-Turbo or Claude 3 Sonnet may be sufficient
  • Consider latency, cost, and capability requirements

Agent Specialization

For complex applications, use specialized agents rather than a single do-everything agent:

  • Create agents with specific roles and expertise
  • Use an agent workforce to coordinate multiple specialized agents
  • Design clear interfaces for agent communication

Conclusion

Agents are the core building blocks of the Agents Hub framework. By understanding the different types of agents, their capabilities, and how to configure them, you can build sophisticated agent-based applications tailored to your specific needs.
